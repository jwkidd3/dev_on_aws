Getting Started with Development on AWS

SDKs, the CLI, and the patterns they share.

Three Ways to Talk to AWS

  • Console — web UI. Great for learning, bad for repeatability.
  • CLI — one command, one API call. Perfect for scripts and CI.
  • SDK — AWS APIs as idiomatic classes / methods in your language

All three hit the same HTTPS endpoints. The console itself is an SDK client.

The Python SDK — boto3

# Amazon Linux 2023 ships Python 3 + pip, but NOT boto3.
# Install once per Cloud9 environment:
pip3 install --user boto3

python3 -c "import boto3; print(boto3.__version__)"
# → 1.x.x

# The AWS CLI is preinstalled:
aws --version

This course uses boto3, the AWS SDK for Python. Lab 1a installs it once; Cloud9's EBS home keeps it across days.

Credential Provider Chain

Every SDK looks here, top to bottom. First match wins.

  1. Explicit credentials passed to the client
  2. AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY env vars
  3. ~/.aws/credentials file (selected by AWS_PROFILE)
  4. Container / EC2 instance / Lambda execution role ← this is what Cloud9 uses

Never hard-code keys. In Cloud9 you get temporary credentials from the instance role automatically — aws sts get-caller-identity just works.

Your First SDK Call

import boto3

# STS — identity check
print(boto3.client("sts").get_caller_identity()["Arn"])

# S3 — list what's in the account
for b in boto3.client("s3").list_buckets()["Buckets"]:
    print(b["Name"])

No credentials in code. boto3 walks the provider chain and picks up the Cloud9 instance's AWS-managed temp credentials automatically.

Common Patterns

  • Paginators — iterate one page at a time without bookkeeping
  • Waiters — block until a resource exists / is ready
  • Retry & backoff — built into every SDK; tune, don't replace
  • Typed exceptions — NoSuchBucket, AccessDenied
  • Presigned URLs — time-limited access to private resources

Regions Matter

  • Most services are regional — a bucket in us-east-1 ≠ eu-west-1
  • Set on the client (region_name="us-east-1") or via AWS_REGION
  • IAM, Route 53, CloudFront are global — no region
  • Misconfigured region is the #1 cause of "resource not found"

AWS Cloud9 — Our Class IDE

  • Pre-installed: AWS CLI, Python, git, jq, Docker, SAM CLI
  • Authenticates via the instance IAM role — no aws configure, no keys on your laptop
