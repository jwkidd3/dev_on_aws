Module 6

Processing Your Storage Operations

Large objects, listing at scale, and presigned URLs.

Operations Worth Knowing

  • Bucket: Create / Delete, Versioning, Lifecycle, Notifications, Policy
  • Object: Put / Get / Delete / Head, Copy, ListV2 (paginated), DeleteObjects (batch)
  • Every SDK call takes a request object and returns a typed response

Large Objects → Multipart

  • Single PUT caps at 5 GB — multipart handles anything larger
  • Use the SDK's Transfer Manager — multipart + parallel + resumable
  • In boto3: s3.upload_file() handles multipart automatically
  • Always add a lifecycle rule to abort incomplete uploads after 7 days

Presigned URLs

url = s3.generate_presigned_url(
    "put_object",
    Params={"Bucket": "acme-uploads-dev",
            "Key": f"uploads/{user_id}/photo.jpg"},
    ExpiresIn=600)
  • Time-limited URL for one specific S3 operation
  • Browser uploads directly to S3 — your API never sees the bytes
  • Treat the URL like a bearer token while it's valid
