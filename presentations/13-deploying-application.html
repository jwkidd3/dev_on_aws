Module 13

Deploying Your Application

DevOps fundamentals, AWS SAM, and shipping safely.

Why Deployment Used to Hurt

  • Big-bang releases every quarter, long regression cycles
  • Manual steps = someone types the wrong command at 2 AM
  • Drift between dev / stage / prod environments
  • Fear of change → fewer deploys → bigger deploys → more risk

Pillars of Modern Delivery

  • Version control everything — code, config, IaC, pipelines
  • CI — every commit builds and tests
  • CD — artifacts promote automatically through environments
  • IaC — environments defined in code; identical across stages
  • Small batches — deploy often, each change small enough to reason about

Why SAM

  • Shorthand for CloudFormation that's serverless-aware
  • Defines Lambdas, APIs, DynamoDB tables, queues in one YAML
  • Local invoke + build + package + deploy from sam CLI
  • Generates a CloudFormation stack you can inspect

A SAM Template

Transform: AWS::Serverless-2016-10-31

Resources:
  ItemsTable:
    Type: AWS::Serverless::SimpleTable
    Properties: { PrimaryKey: { Name: id, Type: String } }

  ItemsFn:
    Type: AWS::Serverless::Function
    Properties:
      Runtime: python3.12
      Handler: handler.handler
      CodeUri: src/
      Environment:
        Variables: { ITEMS_TABLE: !Ref ItemsTable }
      Policies:
        - DynamoDBCrudPolicy: { TableName: !Ref ItemsTable }
      Events:
        Api:
          Type: HttpApi
          Properties: { Path: /items, Method: post }

SAM Workflow

sam build
sam local invoke ItemsFn -e events/post.json
sam local start-api
sam deploy --guided
# Subsequent deploys: sam deploy

First --guided deploy writes samconfig.toml; later deploys reuse it.

Deployment Strategies

  • All-at-once — fast, riskiest
  • Rolling — replace a subset at a time
  • Blue/Green — new stack alongside old; flip traffic
  • Canary / Linear — shift % of traffic gradually; rollback on alarm

Lambda Traffic Shifting

ItemsFn:
  Type: AWS::Serverless::Function
  Properties:
    AutoPublishAlias: live
    DeploymentPreference:
      Type: Canary10Percent5Minutes
      Alarms: [!Ref ErrorsAlarm]

SAM publishes a new version, alias points at old, CodeDeploy shifts 10% for 5 min watching the alarm. Rollback is automatic on alarm fire.
