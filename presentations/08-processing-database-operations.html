Module 8

Processing Your Database Operations

CRUD with DynamoDB from code.

Put + Conditional Write

table = boto3.resource("dynamodb").Table("Items")

table.put_item(
  Item={"pk":"USER#user1","sk":"ITEM#001","title":"Blue widget","price":19.95},
  ConditionExpression="attribute_not_exists(pk)"
)

Fails with ConditionalCheckFailedException if it already exists. Catch and move on — that's idempotency.

Get & Update

table.get_item(Key={"pk":"USER#user1","sk":"ITEM#001"},
               ProjectionExpression="title, price")

table.update_item(
  Key={"pk":"USER#user1","sk":"ITEM#001"},
  UpdateExpression="SET price = :p ADD views :one",
  ExpressionAttributeValues={":p": Decimal("17.50"), ":one": 1},
  ReturnValues="ALL_NEW")

Update verbs: SET · ADD · REMOVE · DELETE.

Query & Scan

  • Query — uses the primary key (required); efficient. Use this.
  • Scan — reads every item and filters client-side. Avoid.
  • Both paginate — check LastEvaluatedKey
  • Filter expressions reduce the response, not the read cost

Query Example

from boto3.dynamodb.conditions import Key

resp = table.query(
  KeyConditionExpression=Key("pk").eq("USER#user1")
                      & Key("sk").begins_with("ITEM#"),
  Limit=25
)
for item in resp["Items"]:
    print(item["title"])

Batch & Transaction

  • BatchGetItem / BatchWriteItem — up to 25 writes / 100 gets per call; retry UnprocessedItems
  • TransactWriteItems — up to 100 actions, all-or-nothing, costs 2× WCU
  • Use transactions only when "two items must change together"

PartiQL

SELECT title, price
FROM  "Items"
WHERE pk = 'USER#user1'
  AND begins_with(sk, 'ITEM#')

SQL-flavored wrapper. Good for ad-hoc; nudges people toward scans without warning.

Best Practices

  • List access patterns first; design key schema second
  • Prefer Query over Scan, every time
  • Use GSIs for "second way to look up the same data"
  • Use TTL to expire ephemeral items (sessions, carts, codes)
  • Alarm on ThrottledRequests — earliest sign of hot keys
