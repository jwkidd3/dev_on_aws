Module 11

Building a Modern Application

Microservices, event-driven design, and Step Functions.

Monolith Pain

  • One codebase, one deploy — a typo in reporting breaks checkout
  • Any team that wants to ship coordinates with every other team
  • All-or-nothing scaling, technology lock-in
  • Long releases → big blast radius → scary rollbacks

Microservices

Small, independently-deployable services that each own one capability and one data store, communicating over APIs or async events.

Independent deploys
Fault isolation
Polyglot
Scale per load

You also accept: distributed-systems problems, mandatory observability, contract testing, eventual consistency.

Decoupling Patterns

  • DDD bounded contexts — one service per business capability
  • Choreography — services publish events, others subscribe (EventBridge, SNS)
  • Orchestration — central coordinator calls services in order (Step Functions)
  • Strangler-fig — move one seam at a time; never "rewrite the monolith"

Event-Driven on AWS

  • EventBridge — central bus, content-based routing
  • SNS — fan-out pub/sub, topic-based
  • SQS — buffering + decoupling, at-least-once delivery
  • Kinesis — ordered, replayable streams
  • DynamoDB Streams — change data capture from your table

AWS Step Functions

  • Serverless state machine — JSON describes states and transitions
  • Task, Choice, Parallel, Map, Wait, Pass/Succeed/Fail
  • Built-in retry, timeout, catch — no boilerplate per Lambda
  • Standard (1 yr, audit history) vs. Express (5 min, high throughput, cheap)

Idempotency Is Mandatory

  • Async + event-driven systems will redeliver
  • Same event applied twice must have the same effect
  • Common: a DynamoDB item keyed on event ID, set to "seen" on first handle
  • AWS Lambda Powertools for Python has a ready-to-use idempotency module

When Serverless Doesn't Fit

  • Heavy per-request CPU — containers cheaper past a threshold
  • Long-lived connections (WebSocket at scale) — ECS/Fargate
  • p99 < 50 ms — cold starts bite
  • State that must live across requests on the same node
