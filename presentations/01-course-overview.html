Developing on AWS

A Three-Day, Code-First Workshop

Module 1 — Course Overview

Welcome

  • 3 days · 7 hands-on labs · one working serverless web app you build yourself
  • Introductory pace — written for developers new to the AWS SDK, not new to programming
  • Lab languages: Python, .NET (C#), Java — pick one and stay with it
  • ~75% of class is keyboard time

What You Will Build

Frontend
Static site on S3
Auth
Cognito user pool
API
API Gateway
Logic
Lambda functions
Data
DynamoDB + S3
Observability
CloudWatch + X-Ray

Every module adds one piece. You'll deploy it all with AWS SAM on Day 3.

Three-Day Shape

  • Day 1 — Foundations. SDK, IAM, S3.
  • Day 2 — Data & Logic. DynamoDB, Lambda, API Gateway.
  • Day 3 — Modern App. Microservices, Cognito, SAM, X-Ray.

Class runs 09:00–17:00. One-hour lunch, two 15-min breaks.

Lab Environment

  • Console: https://kiddcorp.signin.aws.amazon.com/console
  • User: user1, user2, user3… — instructor assigns yours
  • Region: us-east-1 for every lab
  • All resources you create must start with your user prefix: student-user1-…, Items-user1, lab4-user1

Ground Rules

  • Do not touch resources outside your assigned user prefix
  • Never hard-code credentials. Ever. (We'll learn the right way in Module 4.)
  • Clean up at the end of Day 3 — checklist on the last slide of Module 15
  • Ask questions any time — no "hold it to the end"

Introductions — 30 Seconds Each

  • Name and role
  • Primary language (Python / .NET / Java)
  • One thing you want to walk out able to do
