Managing the APIs

Amazon API Gateway — the front door to your serverless app.

What API Gateway Is

The "front door" of your serverless app — the piece that receives HTTP requests from the internet and dispatches them to your backend.

  • In a traditional stack you'd put nginx or an ALB here
  • API Gateway replaces that — fully managed, no servers to patch
  • Handles TLS, auth, throttling, validation, caching, versioning, CORS for you
  • Typical flow: client → API Gateway → Lambda → DynamoDB / S3
  • REST API — feature-rich, more expensive · HTTP API — ~70% cheaper, simpler

Key Components

  • Resource — a path (/items, /items/{id})
  • Method — an HTTP verb on a resource
  • Integration — what runs when the method fires (Lambda, HTTP, AWS, mock)
  • Stage — a deployment (dev, prod); each gets its own URL
  • Authorizer — Cognito, JWT, Lambda, or IAM

Lambda Proxy Integration

  • The default; simplest mental model
  • API GW → Lambda with {path, httpMethod, headers, body, requestContext}
  • Lambda returns {statusCode, headers, body}
  • Mapping templates (VTL) only apply to non-proxy REST integrations

Request Validation

{
  "$schema": "http://json-schema.org/draft-04/schema#",
  "type": "object",
  "required": ["title", "price"],
  "properties": {
    "title": {"type": "string", "maxLength": 80},
    "price": {"type": "number", "minimum": 0}
  }
}

Bad requests rejected with 400 before your Lambda runs. Saves cold starts and money.

CORS & Auth

  • CORS — needed for cross-origin browser calls; "Enable CORS" adds an OPTIONS method
  • Cognito user pool authorizer — validates a JWT and forwards claims
  • JWT authorizer (HTTP API) — any OIDC issuer
  • Lambda authorizer — your own function returns allow/deny + a policy
  • IAM — SigV4 signing; common service-to-service

Stages, Throttling, Logging

  • Changes don't go live until you deploy to a stage
  • Default account limit: 10,000 req/sec, 5,000 burst — region-wide. Override per stage / method.
  • Always rate-limit. Unbounded APIs get abused.
  • Per-stage execution + access logs to CloudWatch

OpenAPI Imports

  • Define the whole API in an OpenAPI / Swagger YAML file
  • Import via console or aws apigateway import-rest-api
  • Diff the file in Git like any other artifact
  • The capstone lab imports a Swagger file — no console clicking
