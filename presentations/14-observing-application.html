Module 14

Observing Your Application

Monitoring, observability, CloudWatch, X-Ray.

Monitoring vs. Observability

  • Monitoring — "is this known metric above/below a threshold?"
  • Observability — "what is this system doing right now?" — investigate without shipping new code
  • Monitoring answers predictable questions. Observability answers the weird ones.

Three Pillars

  • Metrics — numbers over time. Cheap, fast.
  • Logs — text events. Richer, more expensive.
  • Traces — the path of one request across every service it touched, with timings

Good systems use all three. Each answers a different question.

Structured Logging

log.info(json.dumps({
  "request_id": ctx.aws_request_id,
  "user_id":    user_id,
  "action":     "create_item",
  "duration_ms": 42
}))
  • JSON logs → CloudWatch Logs Insights queries them as structured data
  • Always include a correlation/request ID
  • Never log PII or tokens

Alarms That Matter

  • API Gateway 5XXError > 1% for 5 min
  • Lambda Errors > 0 for 2 datapoints
  • DynamoDB ThrottledRequests > 0
  • SQS ApproximateAgeOfOldestMessage > N seconds
  • Alarm on symptoms users notice, not on every metric

AWS X-Ray

  • Distributed tracing across Lambda, API GW, DynamoDB, S3
  • Service map shows the shape of your app visually
  • Trace ID propagates through HTTP headers and Lambda context
  • Pay per trace recorded; sample aggressively in production

Enabling X-Ray

from aws_xray_sdk.core import xray_recorder, patch_all
patch_all()  # auto-instrument boto3, requests, etc.

@xray_recorder.capture("process_item")
def process_item(event):
    xray_recorder.put_annotation("user", event["user"])
    # ...
  • API GW + Lambda: one toggle each (Tracing: Active in SAM)
  • Add AWSXRayDaemonWriteAccess to the function's role
  • Annotations are indexed (filterable); metadata isn't

Four Golden Signals

  • Latency — how long requests take
  • Traffic — how many requests there are
  • Errors — how many are failing
  • Saturation — how full the system is

Start here for every service. Everything else is a sub-question.
