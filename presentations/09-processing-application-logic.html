Module 9

Processing Your Application Logic

AWS Lambda — functions, triggers, packaging, deployment.

What Lambda Is

  • Managed compute that runs your code in response to an event
  • You hand AWS a zip or container; AWS handles servers
  • Pay for ms of execution + invocations — no idle cost
  • Up to 15 min, up to 10 GB of memory
  • First 1M invocations and 400k GB-sec free every month

Anatomy of a Handler

# Python
def handler(event, context):
    user_id = event["requestContext"]["authorizer"]["claims"]["sub"]
    body    = json.loads(event["body"])
    # ... business logic ...
    return {"statusCode": 200,
            "headers": {"Content-Type": "application/json"},
            "body": json.dumps({"id": item_id})}

event shape depends on the trigger. context has request ID, deadline, log group.

Invocation Models

  • Synchronous — caller waits (API Gateway, ALB, CLI)
  • Asynchronous — Lambda queues, retries on failure (S3, SNS, EventBridge)
  • Poll-based — Lambda reads from a stream / queue (SQS, Kinesis, DynamoDB Streams)
  • Error handling differs per model — know which one your trigger uses

Permissions: Two-Sided

  • Execution role — what your code is allowed to do (DynamoDB, S3, etc.)
  • Resource policy — who is allowed to invoke your Lambda (API GW, S3, EventBridge)
  • Always include AWSLambdaBasicExecutionRole for CloudWatch Logs
  • Use a role per function — no shared "AppRole"

Configuration Knobs

  • Memory 128 MB – 10 GB; CPU scales with memory
  • Timeout 1 s – 15 min
  • Ephemeral /tmp up to 10 GB
  • Environment variables — config (table names, log level); use Secrets Manager for actual secrets
  • Concurrency — unreserved (default), reserved (cap), provisioned (pre-warmed)

Versions, Aliases, & Cold Starts

  • Version — immutable snapshot; alias — named pointer to a version
  • Weighted aliases enable canary deploys (95% old / 5% new)
  • Rollback = repoint the alias
  • Cold starts happen on a new environment — usually fine for Python; mitigate with smaller packages or Provisioned Concurrency
  • Init outside the handler — runs once per environment

Anti-Patterns

  • Initializing clients inside the handler
  • Long polling loops — use SQS / EventBridge / Step Functions instead
  • Packaging the dev toolchain into the deploy zip
  • Hard-coded ARNs / endpoints — use env vars or Parameter Store
  • No idempotency — async retries will hurt
