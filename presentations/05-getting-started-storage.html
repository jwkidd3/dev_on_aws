Getting Started with Storage

Amazon S3 — the building block under almost every AWS architecture.

Why S3

  • Object storage — key → bytes, flat namespace
  • 11 nines of durability, 99.99% availability
  • Scales from 1 KB to petabytes with zero tuning
  • Pay only for bytes stored, requests, and data transferred out

Vocabulary

  • Bucket — globally-unique named container
  • Object — bytes you store (up to 5 TB)
  • Key — object name; looks like a path, isn't one
  • Prefix — part of a key before /, used for listing
  • ETag — content hash returned on PUT / GET
  • Versioning — keep prior revisions of a key

Security Defaults

  • Encryption at rest on by default (SSE-S3) since 2023
  • Block Public Access on by default for new buckets
  • Access controlled by IAM policies (principal) + bucket policies (resource)
  • Upgrade to SSE-KMS when you need audit, rotation, cross-account grants

Consistency & Exceptions

  • Strongly consistent for PUT / GET / DELETE since Dec 2020
  • NoSuchBucket / NoSuchKey — wrong name or region
  • AccessDenied — IAM, bucket policy, or BPA blocked it
  • SlowDown — SDK retries with backoff automatically
