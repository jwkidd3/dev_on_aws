Module 7

Getting Started with Databases

Amazon DynamoDB — the key/value NoSQL store behind most serverless apps.

Why DynamoDB

  • Managed — no servers, patches, or backups to run yourself
  • Single-digit millisecond reads and writes at any scale
  • On-demand or provisioned capacity — your choice
  • Native integrations with Lambda, Streams, Kinesis

SQL Instincts to Unlearn

  • No JOIN — model access patterns, not entities
  • No schema — items in one table can have different attributes
  • No ad-hoc queries — query by primary key or an index
  • Transactions exist (max 100 items) but are not the default

Keys

  • Every table has a primary key
  • Simple: partition key only
  • Composite: partition key + sort key
  • Partition key picks the shard; sort key orders items in a partition
  • High-cardinality partition keys (user IDs) are safe — low-cardinality ones get "hot"

Item Anatomy

{
  "pk":     "USER#user1",
  "sk":     "ITEM#2026-04-16T14:02Z",
  "title":  "Blue widget",
  "price":  19.95,
  "tags":   ["blue", "widget"],
  "inStock": true
}

Max item size 400 KB — store blobs in S3 and keep the key in DynamoDB.

Indexes

  • GSI — any partition + sort key, async replication, eventual reads
  • LSI — same partition key as table, different sort, set at create time
  • Indexes cost capacity — only build the ones your access patterns need

Connecting

# Python — low level
ddb = boto3.client("dynamodb")

# Python — resource (typed values)
table = boto3.resource("dynamodb").Table("Items")

# Java v2 — enhanced (object mapper)
DynamoDbEnhancedClient enh =
   DynamoDbEnhancedClient.builder()
       .dynamoDbClient(DynamoDbClient.create()).build();

Pick one layer per module of your code. Mixing them gets confusing fast.

Common Exceptions

  • ResourceNotFoundException — table or index missing
  • ConditionalCheckFailedException — your ConditionExpression didn't match
  • ProvisionedThroughputExceededException — throttled; back off
  • ValidationException — wrong types or missing keys
