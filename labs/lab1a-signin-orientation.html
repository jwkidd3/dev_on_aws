Sign In & Create Your Cloud9 Environment

Module 4 — Permissions 45 min

AWS Account Access 3 min

  • Console: https://kiddcorp.signin.aws.amazon.com/console
  • Username: user1, user2, … — instructor assigns
  • Password: handed out separately
  • Region: us-east-1

Objectives 3 min

  • Sign into the class account as your assigned user
  • Create your own Cloud9 IDE (m5.large, SSH)
  • Confirm the AWS CLI + SDKs work from the Cloud9 terminal
  • Learn the "editor for files, terminal for commands" convention used all week

Step 1 — Sign In 3 min

  1. Open the console URL in an incognito window
  2. Enter your assigned username (e.g., user1) and password
  3. Top-right region selector → US East (N. Virginia) us-east-1

Step 2 — Create Your Cloud9 Environment 10 min

  1. Console search → Cloud9 → Create environment
  2. Name: dev-on-aws-user1 (replace user1 with your assigned user)
  3. Environment type: New EC2 instance
  4. Instance type: m5.large
  5. Platform: Amazon Linux 2023
  6. Timeout: 30 minutes (saves cost when idle)
  7. Connection: Secure Shell (SSH) ← important, do not pick SSM
  8. Network settings: default VPC, any public subnet in us-east-1
  9. Create — provisioning takes ~3 minutes

Step 3 — Open Cloud9 & Smoke Test 8 min

  1. Wait for environment status to go from Creating to Available
  2. Click Open — the IDE loads in a new browser tab
  3. Bottom pane is the terminal. Run:
aws --version          # expect aws-cli/2.x
python3 --version
git --version
jq --version
aws sts get-caller-identity
# Arn: arn:aws:iam::...:user/user1   ← confirms your IAM identity

Step 4 — Clone the Course Repo 4 min

Each lab has supporting files (handlers, SAM templates, helper scripts) on GitHub. Clone once, copy the supporting files into your working directory, reuse all week.

cd ~/environment
git clone https://github.com/jwkidd3/dev_on_aws
cp -r dev_on_aws/labs/files ./dev-on-aws

# Verify — every MISS below means subsequent labs will break, raise hand
cd ~/environment/dev-on-aws
for f in lab1/smoke_test.py lab3/seed.py lab4/handler.py \
         lab4/lambda-perms.json lab6/add_methods.sh \
         lab7/python/handler.py lab7/template.yaml; do
  [ -f "$f" ] && echo "OK   $f" || echo "MISS $f"
done

Every line should say OK. If any say MISS, the clone didn't land — raise your hand before continuing. ~/environment is Cloud9's persistent EBS home, so you only run this once.

Step 5 — Editor vs. Terminal (Class Convention) 5 min

Throughout the course:

  • Source files — already on disk from Step 4. Open in the Cloud9 editor to read, edit, or adjust placeholders (e.g., replace ACCT with your account ID). Save with ⌘S / Ctrl+S.
  • Commands to run — short snippets (≤ ~5 lines) pasted into the Cloud9 terminal. Anything larger lives as a file (see above) and is invoked by name.
  • Never paste multi-line source into a shell prompt — terminals mangle indentation, quotes, and tabs.

Step 6 — Prefix Convention 5 min

Every resource you create today must start with your user prefix:

  • S3 buckets: student-user1-*
  • DynamoDB tables: Items-user1
  • Lambda functions: lab4-user1, lab7-user1
  • IAM policies you create: Student*-user1

The policies attached to your user1 IAM user enforce this prefix. Off-prefix actions return AccessDenied.
